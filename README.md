# Personal Finance Reasoning Dataset

A synthetic instruction-tuning dataset designed to teach language models to reason through personal finance and investing decisions using the mental frameworks from classic books in the genre. The goal is not recall of book content but principled reasoning: the model should apply frameworks to novel situations it has never seen.

## Source Books

Principles were extracted from the following books:

- *The Psychology of Money* — Morgan Housel
- *Rich Dad Poor Dad* — Robert Kiyosaki
- *The Richest Man in Babylon* — George S. Clason
- *The Intelligent Investor* — Benjamin Graham
- *Think and Grow Rich* — Napoleon Hill

## Dataset Statistics

| Split | Examples |
|-------|----------|
| Train | 651 |
| Validation | 73 |
| **Total** | **724** |

| Example Type | Count | Description |
|---|---|---|
| Type A — Applied Q&A | 324 | Novel scenario question answered using a book principle |
| Type B — Chain-of-Thought | 324 | Financial dilemma with step-by-step reasoning that names and applies a principle |
| Type C — Cross-book Contrast | 76 | Scenario where two principles from different books compete; model selects the more applicable one |

## Data Format

Each example is a JSON object with a `messages` array following the standard chat format, compatible with HuggingFace `apply_chat_template`.

```json
{
  "messages": [
    {
      "role": "user",
      "content": "Why does my friend choose to live paycheck to paycheck while I save aggressively for retirement? Aren't they being irresponsible with their money?"
    },
    {
      "role": "assistant",
      "content": "<think>\nFinancial behaviors are shaped by personal history, not just knowledge. Someone raised in financial instability may rationally prioritize immediate needs. This connects to the 'No One's Crazy' principle — every financial decision makes sense from inside the person's lived experience, even if it looks irrational from outside...\n</think>\n\nYour friend isn't necessarily being irresponsible — they may be operating from a completely different set of experiences and constraints than you..."
    }
  ]
}
```

### Key structural feature: `<think>` blocks

Every assistant turn contains a `<think>...</think>` block before the final answer. This block contains explicit step-by-step reasoning that names and applies the relevant financial principle. This is intentional: training on this structure teaches the model to reason visibly through a framework before answering, rather than producing generic financial advice.

### Fields

| Field | Type | Description |
|---|---|---|
| `messages` | array | Chat turns in `[{role, content}, ...]` format |
| `messages[0].role` | string | Always `"user"` |
| `messages[0].content` | string | The scenario question or financial dilemma |
| `messages[1].role` | string | Always `"assistant"` |
| `messages[1].content` | string | `<think>...</think>` reasoning block followed by final answer |

## Generation Process

This dataset was generated synthetically in three stages:

1. **Principle extraction** — Each source book was parsed and its core principles were extracted as structured JSON (name, description, application method, underlying mental model) using GPT-4o-mini.

2. **Example generation** — For each principle, three example types were generated using GPT-4o (Types B and C) and GPT-4o-mini (Type A). Prompts explicitly required the model to embed `<think>` blocks and to apply the principle by name without directly quoting the source book.

3. **Quality filtering** — Examples were dropped if the assistant response was under 200 characters, lacked a `<think>` block, was a duplicate (matched another example's first 100 characters), or contained a verbatim quote longer than 30 words.

## Intended Use

This dataset is intended for **supervised fine-tuning (SFT)** of instruction-following language models, specifically to steer reasoning style toward personal finance and wealth-building frameworks. It is well-suited for parameter-efficient methods (LoRA, QLoRA) on smaller models (1B–7B parameters).

It is not intended for models to memorize the books' content. The eval criterion for a well-trained model is whether it applies frameworks correctly to novel scenarios, not whether it can reproduce book passages.

## Limitations

- **Synthetic data bias**: All examples were generated by GPT-4o/GPT-4o-mini. The quality and diversity of reasoning is bounded by those models' understanding of the source material.
- **Coverage is uneven**: Books with clearly structured chapters (e.g., *Rich Dad Poor Dad*) produced more principles and examples than books with narrative/parable formats (e.g., *Think and Grow Rich*).
- **English only**: All content is in English.
- **No financial advice**: This dataset is for ML research and model training. Nothing here constitutes professional financial advice.
